FAQ

Frequently asked questions

Everything you need to know about turning your commits into build in public posts.

Shipublic is an open source local first tool that turns your git commits and code diffs into ready to publish build in public posts. It connects to your GitHub account, analyzes diffs with local Ollama models, and produces clean, story style updates for LinkedIn, X, and other platforms.
Yes. AGPL 3.0 licensed, no usage limits, no cap on commits analyzed or posts generated. You self host the backend, frontend, database, and Ollama runtime. The only cost is your hardware and electricity.
Not for analysis. Diffs and commit messages are sent only to your local Ollama instance. Posts are stored in your local Postgres. The only outbound traffic is the GitHub API to fetch your commits, and only when you choose to publish.
By default, qwen2.5 coder 32b reads diffs and produces a structured developer summary. qwen3 27b takes that summary and rewrites it as a slightly engaging, non cringe build in public post. Both models are configurable per project.
Backend: NestJS 11 with Fastify adapter, Prisma ORM, PostgreSQL, Redis 7, BullMQ for job queues, Ollama for local LLM inference. Frontend: Next.js 15 with TypeScript and CSS Modules. Lucide icon system. Everything containerized with Docker Compose for one command deployment.
Toggle auto sync per project. Shipublic registers a GitHub webhook (or git hook for self hosted repos). Every push enqueues a BullMQ job that fetches the new commits, runs the hybrid Ollama pipeline, and lands a draft post in your calendar ready for review.
The backend, Postgres, and Redis run on any server with 2GB RAM. Ollama with qwen2.5 coder 32b benefits from a GPU with at least 24GB VRAM, but smaller quantized models run on CPU or modest GPUs. Use OLLAMA_NUM_GPU and OLLAMA_MAX_LOADED_MODELS to tune for your machine.
